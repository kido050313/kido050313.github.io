关于forEach for...of for...in

发表于 2019-11-06 | 分类于 javascript |
字数统计: 971

  项目中常常用到遍历，对于forEach, for…of, for…in一直不太懂到底有什么区别，什么时候该用哪一个。今天抽了点时间打开mdn看了一下三者的描述。没想到上面其实已经说的很清楚了，大概记录一下自己看了之后的想法。

forEach

  forEach() 方法对数组的每个元素执行一次提供的函数。

   forEach() 除了抛出异常以外，没有办法中止或者跳出循环，break,return 都不可以。
   如果需要提前终止循环，可以使用 for, for…of/for…in， every(), some(), find(), findIndex()等方法。
  其中forEach()的返回值是undefined,这里我比较疑惑，查了资料，觉得是this指向问题，但是具体的此时的我尚未知晓，所以这里插一个flag，有时间再深入研究。如果有知道的大佬愿意科普一下就多谢了。
  总之，如果没有特殊要求，forEach()是足够了，但目前的话更推荐for…of。

for…of

  for…of语句在可迭代对象（包括 Array，Map，Set，String，TypedArray，arguments 对象等等）上创建一个迭代循环，调用自定义迭代钩子，并为每个不同属性的值执行语句。

  在我看来，for…of就是forEach()的升级版，可迭代对象更多，而且与forEach()不同，for…of可以正确响应break ,throw continue, return等语句。

for…in

   for…in语句以任意顺序遍历一个对象的除Symbol以外的可枚举属性。for…in 循环只遍历可枚举属性（包括它的原型链上的可枚举属性）
   如果需要处理有key-value的数据（比如属性用作“键”），需要检查其中的任何键是否为某值的情况时，推荐用for … in。迭代数组不建议使用for…in，因为使用for…in进行数组遍历时不一定按次序访问元素。

for…of与for…in不同之处

它们之间的主要区别在于它们的迭代方式：
  for…in 语句以任意顺序迭代对象的可枚举属性。
  for…of 语句遍历可迭代对象定义要迭代的数据。

总结

  1. for…of与forEach不同的是，使用break,return等语句时，可以正确中止程序，同时for…of可迭代的对象更多，我理解为for…of作为forEach的升级版。

  2. for-of循环不支持普通对象，但如果你想迭代一个对象的属性，你可以用for-in循环（这也是它的本职工作）或内建的Object.keys()方法。如果你想迭代数组，推荐使用for…of/forEach,不推荐for…in。
